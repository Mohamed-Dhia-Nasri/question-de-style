## Convention refinements (plan amended)

- **Tenant column style (seam 14):** tenant-owned tables use `foreignId('tenant_id')->constrained('tenants')->cascadeOnDelete()` + an index (e.g. `monitoring_settings` migration :23), not a bare `unsignedBigInteger`. → `keyframes` and `content_transcripts` migrations amended.
- **Redundant index (seam 14):** the planned extra `index(['owner_type','owner_id'])` on `keyframes` is redundant — the unique `(owner_type, owner_id, ordinal)` already serves leftmost-prefix lookups; `review_actions` carries exactly one composite index. → dropped.
- **Partial-unique verdict (seam 14):** partial indexes here exist only for the nullable-XOR target pair on `recognition_detections`; a both-NOT-NULL polymorphic pair (the `review_actions` style) takes plain composite indexes. Plain `unique(owner_type, owner_id, ordinal)` is the house style. Ids are bigint `$table->id()` everywhere.
- **`recordOperation` concrete shape (seam 2):** `ProviderCallRecorder` has NO shared private row-writer — `recordCompletion` and `recordFailure` each inline their own `ProviderCall::query()->create()`. The consistent implementation is a **third inline create** (context base fields + `http_status`/`response_bytes`/`rate_limit`/`timings.request_ms` from the `ProviderResponse`, `outcome = Success`, `result_count = accepted_count = $resultCount`), then reuse the existing internals verbatim: `$this->markSuccess($context, CallOutcome::Success)`, `$this->raiseDurationAlertIfNeeded($context)`, and `$this->sampler->maybeSample(..., $response)` — `maybeSample`'s 4th param is already typed `ProviderResponse`, so no fabricated batch anywhere. Signature: `recordOperation(CallContext $context, ProviderResponse $response, int $resultCount): ProviderCall`. → Task 13 amended with this exact shape.
- **Test-helper names (seam 7a/7b):** `TestCase::$defaultTenant` confirmed. `RecognitionPipelineTest` builders are `imagePost()`/`reel()` (media_urls hardcoded to `https://93.184.216.34/img-1.jpg`), fakes are `fakeVision(array)`/`fakeAudio(...)` (media-host fake folded into `fakeVision`; video tests write inline `Http::fake`), and the direct call helper is `enrich(ContentItem $content, string $correlationId = 'corr-1')`. → Task 7's test amended to use `reel()` + `enrich()`.
- **Stages assertions (seam 8):** per-key only, no whole-array asserts. One ordering-sensitive spot: `EnrichmentPipelineTest.php:176-178` asserts which stage keys exist up to a vision failure — the new `transcript` stage (runs BEFORE recognition) WILL be present there; `keyframes` (after recognition) will not. → noted in Task 13.
